# FRONTEND_REDESIGN_REPORT

## Проблемы старого UI
1. Главный экран перегружен техническими блоками.
2. "Состояние корпуса" не помогает пользователю выполнить задачу.
3. "Логика режимов" дублирует информацию из выбора режима.
4. Правая колонка отвлекает от вопроса и ответа.
5. Технические детали должны быть скрыты в "О системе".
6. Пользовательский путь должен быть: вопрос -> режим -> ответ -> источники.
7. Интерфейс должен говорить языком пользователя, а не языком разработчика.

## Second redesign: product-focused UI
1. Что было не так в предыдущем редизайне.
- Экран всё ещё выглядел как техническая демо-панель.
- Ответ и источники были разделены на отдельные большие блоки и теряли приоритет.
- Правая колонка визуально перетягивала внимание.

2. Что убрано.
- Убраны отдельные крупные карточки для разрозненных состояний на главном экране.
- Убрана перегруженная вертикальная форма режимов.
- Убран постоянный отдельный блок уточнения как самостоятельный экранный модуль.

3. Что перенесено в "О системе".
- Техническая конфигурация corpus/retrieval/storage/embeddings вынесена в `details` внутри helper-панели.
- Главный рабочий сценарий освобождён от архитектурных деталей.

4. Как теперь устроен основной assistant panel.
- Реализован единый `assistant panel`:
  - Question composer
  - Controls row (версия + segmented modes)
  - Submit/status
  - Answer states (empty/loading/error/success)
  - Inline clarification (по требованию backend)
  - Sources под ответом

5. Как теперь устроена правая колонка.
- Единая компактная helper-панель:
  - Примеры вопросов
  - Недавние запросы (максимум 4)
  - Компактный `О системе` (свёрнут по умолчанию)
- Добавлена ссылка `Открыть историю` для полного списка на `/history`.

6. Как теперь отображается ответ.
- Ответ показывается сразу под формой в том же assistant panel.
- Есть компактные badges: версия, режим, policy.
- Сохранены states: empty/loading/error/success.
- После генерации выполняется smooth-scroll к блоку ответа при необходимости.

7. Как теперь отображаются источники.
- Источники идут сразу под ответом, внутри assistant panel.
- При отсутствии источников выводится компактный muted текст.
- Кнопка `Показать ещё` появляется только при `sources.length > visibleLimit`.
- Для `short/detailed` скрываются tutorial/supplementary источники.

8. Как теперь работает clarification block.
- `Нужно уточнение` не показывается постоянно.
- Блок рендерится inline и раскрывается только если backend вернул `clarification`.
- В остальных сценариях остаётся скрытым.

9. Как теперь отображаются ошибки.
- User-facing error компактный и не занимает половину экрана.
- Технические детали скрыты в `details` по умолчанию.
- Raw backend error выводится в `pre` внутри technical details.

10. Что не менялось.
- corpus
- embeddings
- BAAI/bge-m3
- reindex
- database
- Alembic migrations

## Дополнительные технические заметки
- Для корректной работы embeddings в проекте ожидается `EMBEDDING_PROVIDER=local`.
- `.env` не изменялся; `.env.example` уже содержит корректное значение.

## Результат второго редизайна
- Главный экран: цельный assistant workspace, а не набор разрозненных форм.
- Фокус: вопрос -> ответ -> источники.
- Правая колонка: помогает, но не доминирует.
- История `/history`: статус `Ответ сформирован` вместо акцентно-зелёного `Успешно`.
