# Краткая шпаргалка по psql для PostgreSQL 16

## Назначение

Этот файл даёт практический минимум по `psql`, который нужен в tutorial-ответах: как подключиться, как быстро понять "куда подключились", как посмотреть объекты, как выполнить скрипт и как не путать SQL с meta-командами клиента. Для новичка это основной инструмент первичной диагностики.

## Когда использовать

- нужно подключиться к PostgreSQL по хосту, порту и пользователю;
- требуется проверить, в какой базе и под какой ролью выполняются команды;
- нужно быстро показать таблицы, индексы, схемы, роли;
- требуется выполнить `.sql`-файл;
- пользователь получил ошибку подключения и не понимает причину;
- нужно импортировать/экспортировать CSV без отдельного ETL.

## Простое объяснение

`psql` умеет две разные категории команд. Первая — обычный SQL (`SELECT`, `CREATE TABLE`, `ALTER ROLE`), эти команды отправляются на сервер PostgreSQL. Вторая — meta-команды `psql`, которые начинаются с обратного слеша (`\l`, `\dt`, `\timing`) и обрабатываются самим клиентом.

Если смешивать эти категории, появляются ложные ошибки. Например, `\dt;` с точкой с запятой воспринимается иначе, чем SQL-запрос; а `SELECT ...` без `;` в интерактивном сеансе может не выполниться сразу. Поэтому в обучающих ответах важно явно говорить: это SQL-команда или meta-команда.

## Предварительные условия

- установлен клиент `psql` и доступен в `PATH`;
- известны параметры подключения: host, port, database, user;
- пользователь знает пароль или настроен `PGPASSWORD`/`.pgpass`;
- целевой сервер реально запущен и принимает TCP-подключения;
- для `\copy` и `\i` есть доступ к файлам на стороне клиента.

## Минимальный рабочий пример

```bash
psql -h 127.0.0.1 -p 5432 -U app_user -d app_db
```

```sql
SELECT current_database(), current_user;
```

```text
\l
\c app_db
\dt
\d app_table
\du
\dn
\di
\x
\timing
```

```bash
psql -h 127.0.0.1 -p 5432 -U app_user -d app_db -f init.sql
```

## Пошаговый алгоритм

1. Подключись к серверу через `psql -h -p -U -d`.
2. Сразу проверь контекст: `SELECT current_database(), current_user;`.
3. Покажи список баз `\l` и при необходимости переключись `\c`.
4. Просмотри объекты: `\dt`, `\d table_name`, `\di`, `\dn`, `\du`.
5. Включи удобный режим чтения результата: `\x` для expanded output.
6. Включи измерение времени `\timing`, если обсуждается производительность.
7. Для записи вывода в файл используй `\o report.txt`, затем верни вывод в stdout командой `\o`.
8. Для запуска SQL-скрипта из текущей сессии используй `\i script.sql`.
9. Для клиентского CSV-импорта/экспорта применяй `\copy`, когда файл находится на машине клиента.
10. Для неинтерактивного выполнения используй `psql ... -f script.sql`.

## Как проверить результат

```sql
SELECT current_database(), current_user;
```

```text
\l
\dt
\d app_table
\timing
```

```text
\copy app_table(col1, col2) FROM 'data.csv' WITH (FORMAT csv, HEADER true)
\copy (SELECT col1, col2 FROM app_table ORDER BY col1) TO 'out.csv' WITH (FORMAT csv, HEADER true)
```

- команда `current_database()` должна вернуть ожидаемую базу;
- `current_user` должен совпадать с целевой ролью;
- meta-команды должны показывать именно те объекты, которые ожидаются;
- SQL-файл выполняется без ошибок и оставляет проверяемый результат.

## Типовые ошибки

- Путать SQL и meta-команды `psql`.
- Подключиться не к тому серверу (локальный PostgreSQL вместо remote/Docker).
- Проверять схему/таблицу до `\c` в нужную базу.
- Игнорировать `current_database()` и `current_user` при диагностике прав.
- Пытаться использовать серверный `COPY`, когда нужен клиентский `\copy`.
- Выполнять `\i` из каталога, где нет файла, и считать это ошибкой SQL.

## Безопасность и ограничения

- Не передавай пароль в истории shell в открытом виде; лучше использовать `.pgpass`.
- `\copy` работает с файлами клиента, это удобно, но требует аккуратности с чувствительными данными.
- `psql` в production следует запускать с осознанными правами и явной целевой базой.
- Этот файл — учебная шпаргалка; точный синтаксис и редкие флаги смотри в official docs.

`psql` — это одновременно SQL-клиент и набор meta-команд для быстрой диагностики.
