[E.10. Release 17 # / E.10.1. Overview #] New memory management system for VACUUM , which reduces memory consumption and can improve overall vacuuming performance.
[E.10. Release 17 # / E.10.1. Overview #] New memory management system for VACUUM , which reduces memory consumption and can improve overall vacuuming performance.
[E.10. Release 17 # / E.10.1. Overview #] New SQL/JSON capabilities, including constructors, identity functions, and the JSON_TABLE() function, which converts JSON data into a table representation.
[E.10. Release 17 # / E.10.1. Overview #] New SQL/JSON capabilities, including constructors, identity functions, and the JSON_TABLE() function, which converts JSON data into a table representation.
[E.10. Release 17 # / E.10.1. Overview #] Various query performance improvements, including for sequential reads using streaming I/O, write throughput under high concurrency, and searches over multiple values in a btree index.
[E.10. Release 17 # / E.10.1. Overview #] Various query performance improvements, including for sequential reads using streaming I/O, write throughput under high concurrency, and searches over multiple values in a btree index.

[E.10. Release 17 # / E.10.2. Migration to Version 17 #] A dump/restore using pg_dumpall or use of pg_upgrade or logical replication is required for those wishing to migrate data from any previous release. See Section 18.6 for general information on migrating to new major releases.
