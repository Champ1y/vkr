[E.4. Release 18 # / E.4.1. Overview #] PostgreSQL 18 contains many new features and enhancements, including:
[E.4. Release 18 # / E.4.1. Overview #] An asynchronous I/O (AIO) subsystem that can improve performance of sequential scans, bitmap heap scans, vacuums, and other operations.
[E.4. Release 18 # / E.4.1. Overview #] pg_upgrade now retains optimizer statistics.
[E.4. Release 18 # / E.4.1. Overview #] Support for "skip scan" lookups that allow using multicolumn B-tree indexes in more cases.
[E.4. Release 18 # / E.4.1. Overview #] uuidv7() function for generating timestamp-ordered UUIDs .
[E.4. Release 18 # / E.4.1. Overview #] Virtual generated columns that compute their values during read operations. This is now the default for generated columns.
[E.4. Release 18 # / E.4.1. Overview #] OLD and NEW support for RETURNING clauses in INSERT , UPDATE , DELETE , and MERGE commands.
[E.4. Release 18 # / E.4.1. Overview #] Temporal constraints, or constraints over ranges, for PRIMARY KEY , UNIQUE , and FOREIGN KEY constraints.
[E.4. Release 18 # / E.4.1. Overview #] The above items and other new features of PostgreSQL 18 are explained in more detail in the sections below.
[E.4. Release 18 # / E.4.2. Migration to Version 18 #] A dump/restore using pg_dumpall or use of pg_upgrade or logical replication is required for those wishing to migrate data from any previous release. See Section 18.6 for general information on migrating to new major releases.
